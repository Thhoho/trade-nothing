# Trade Nothing v10.0 — 对抗式投研数据获取板块架构设计规范 (Adversarial Data Spec)

> **版本号**: v0.9.1 · **阶段**: 架构设计规范 (v10.0 重构预研)  
> **核心哲学**: 我们不是要完全还原真实世界（这是平庸且昂贵的防守），我们是要在“不完整事实”构成的弥天大雾中，以极低的成本寻找**共识的盲区**与**逻辑的硬伤**。

---

## 1. 数据板块的第一性原理 (First Principles)

### 1.1 为什么传统的“大而全”数据集成是错的？
1. **防守型陷阱 (Defensive Trap)**：购买昂贵的商业数据源（如 Bloomberg、Wind）只能获取“大众已知的落后事实”。这种数据充沛的地方，超额收益（Alpha）已经降为零。
2. **完全信息幻觉 (Illusion of Perfect Info)**：试图完全还原真实世界在数学上是不可行的（有噪声且滞后）。投研的本质是 **非对称信息下的决策博弈**，是用 10% 的“关键边缘事实”撬动 90% 的“未来概率重新定价”。
3. **高摩擦与高成本**：个人研究者或轻量智能体无法承担高昂的 API 费用，且高频的全量数据抓取极易面临反爬封禁。

### 1.2 Trade Nothing 的数据获取板块到底需要什么？
在我们的**对抗式辩论与形式论证框架 (Dung 1995)** 中，数据不是用来做大屏展示的，它们只扮演两种角色：
* **平庸共识的锚点 (Consensus Anchors)**：用于确定市场当前相信什么，从而在提示词中划定“平庸共识禁区”（Clichés）。
* **质证攻击的刺针 (Micro-Fact Needles / Attacks)**：作为 **侦探 (Detective)** 和 **审问者 (Inquisitor)** 在辩论循环中用于发起 `attack` 或 `support` 的微观断言与论据。

因此，**我们的数据层不需要“完美无缺”，它只需要“可被验证的碎裂事实串联”。**

## 2. 三层渐进式实用数据架构 (Tri-Tier Pragmatic Data Architecture)

为了在可用性、获取难度与运行成本之间取得最优解，我们将数据获取板块划分为三层结构：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       1. 市场/宏观锚点数据 (Anchor Data)                  │
│   - 作用: 提供基础物理限制与常识约束 (Price, PE, US10Y, VIX)             │
│   - 来源: 免 Key / 免费且极度稳定的公开 API (yfinance, FRED, 腾讯/新浪HQ)  │
└────────────────────────────────────┬────────────────────────────────────┘
                                     │
                                     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                       2. 共识与预期偏离数据 (Consensus & Gap Data)       │
│   - 作用: 锁定市场的“中位数预期”，寻找“共识的墙”                        │
│   - 来源: 机构一致预期、Polymarket 预测事件赔率、社交媒体舆情偏离度         │
└────────────────────────────────────┬────────────────────────────────────┘
                                     │
                                     ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                    3. 局部草根证伪事实 (Micro-Facts & Catalysts)        │
│   - 作用: 注入“微观非共识刺针” (Custom scrapers, Customs data, Pricing)  │
│   - 来源: 垂直海关数据、产品现货价、GitHub/开源社区活跃度、专利申请公示      │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.1 第一层：市场与宏观锚点 (Tier 1: Anchor Data)
* **目的**：为系统提供最基础的“物理常识”，防止智能体进行脱离金融现实的幻觉推演。
* **推荐接入源**：
  * **yfinance (全球资产/宏观基准)**：用于提取标的实时价格、10年期美债收益率 (`^TNX`)、恐慌指数 (`^VIX`)、布伦特原油、黄金价格。无需 API Key，极其稳定。
  * **腾讯/新浪 HQ (国内市场实时报价)**：单只股票极简的行情数据，响应时间 < 50ms。
  * **FRED (圣路易斯联储宏观数据库)**：免费且官方的全球宏观指标，如美国 CPI、联邦基金利率、利差指标。
* **设计原则**：强缓存，TTL = 1小时。仅在分析开始（Phase 1）时拉取一次。

### 2.2 第二层：市场共识与预期偏离 (Tier 2: Consensus & Gap Data)
* **目的**：锁定敌人的防线。只有知道大众普遍预期公司今年增长 15%，侦探才能去论证“因为 HJT 出口超预期，实际会增长 40%”；审问者才能去质证“因为低温银浆短缺，增长可能只有 5%”。
* **推荐接入源**：
  * **分析师一致预期 (Consensus Estimates)**：抓取东方财富或同花顺的公开个股研报汇总页面，提取核心共识数据：未来 1-2 年的 **EPS 预测中位数**、**营收增长率中位数** 及 **机构评级分布**。
  * **预测市场 odds (Polymarket Gamma API)**：在宏观议题、降息预期、特定行业政策（如电动汽车补贴、加密监管）上，直接获取真实资金博弈出的客观概率，作为辩论的先验概率。
  * **舆情偏离指数 (Narrative Arbitrage Indicator)**：爬取雪球/Reddit 热门讨论发帖频次及情感倾向（使用极轻量的规则或本地词典判定），对比分析师覆盖度。若社交讨论爆表但研报零覆盖，即存在强叙事套利机会。

### 2.3 第三层：微观碎裂事实与催化剂 (Tier 3: Micro-Facts & Alternative Data)
* **目的**：提供用于发起逻辑攻击（Dung attack）的“硬子弹”。
* **推荐接入源**：
  * **行业产品现货价格 (Spot Market Prices)**：对于周期性品种（如碳酸锂、光伏硅片/浆料、芯片晶圆、海运集装箱指数 CCFI），通过极其轻量的 API 或免登录 HTML 爬虫，抓取每周现货指导价。
  * **海关月度进出口数据 (Customs Export Analysis)**：月度出口数据比公司季报领先 1.5 到 2 个月。通过抓取海关总署公开的月度特定 HS 编码（如逆变器 HS 8504409090）出口数据，可直接获得技术或出海企业最真实的景气拐点。
  * **开发者生态与专利 (Technology Trace)**：对于科技股和硬科技标的，抓取 GitHub Star/Issues 增长率、技术专利公示网。当一个新兴 AI 库的 Star数在 1 个月内飙升 10 倍，这就是难以掩盖的爆发事实（Vision Mode 的极佳佐证）。

## 3. 对抗式数据网关的设计模式 (Engine Integration)

在 Trade Nothing 中，我们不执行昂贵的 ETL 同步，而是采用 **“数据按需转换断言 (Data-to-Claim Mapping)”** 模式：

```
                             Raw Data
             (例如: 宁波海关 Q1 组件出口额 YoY +35%)
                                │
                                ▼
         ┌─────────────────────────────────────────────┐
         │       1. 断言模板化 (Claim Template)        │
         │ "海关数据显示, [Symbol] 核心出口额同比+35%, │
         │  大幅偏离分析师一致预期的 15% 增速门槛。"   │
         └──────────────────────┬──────────────────────┘
                                │
                                ▼
         ┌─────────────────────────────────────────────┐
         │       2. 论点分类器 (Claim Classifier)       │
         │ - 归类为: [Detective: Audit Node]          │
         │ - 可信度度量 (Freshness & Integrity): 0.90   │
         └──────────────────────┬──────────────────────┘
                                │
                                ▼
         ┌─────────────────────────────────────────────┐
         │       3. 注入对抗辩论上下文 (Dung Solver)    │
         │  Detective 利用该节点攻击 System:Cliche     │
         └─────────────────────────────────────────────┘
```

在 `scripts/data_providers.py` 中，所有的数据提供商不直接输出 `dict` 或 `dataframe`，而是继承自一个统一的断言生成类：

## 4. 共识传导与非对称博弈窗口模型 (Consensus Transmission & Information Arbitrage Model)

为了落实“在恰当的时机博取投资机会（不慢、不快、不接最后一棒）”的工程化表达，Trade Nothing 建立了一套**“共识传导阶段分类器”**。我们通过对 Dung 形式化论证框架中已被接受的节点分布进行结构化分类，定量评估当前的博弈窗口。

### 4.1 传导阶段的数学特征与博弈策略

我们定义三类不同生命周期的论证节点：
* **$N_{raw}$（微观事实节点）**：源自海关 HS 数据、现货价、单笔招标等局部感知到的“物理硬事实”。
* **$N_{prof}$（专业共识节点）**：源自卖方一致预期 EPS、深度行业投研报告、专家内部纪要。
* **$N_{hype}$（大众平庸共识节点）**：源自雪球/论坛大V情绪偏离、极高估值倍数（PE）、散户讨论热度。

通过量化分析 $N_{raw}$、$N_{prof}$、$N_{hype}$ 的分布，我们划分出四个清晰的非对称博弈阶段：

| 传导阶段 | 核心特征描述 | Expectation Gap (EGI) 状态 | 估值溢价与筹码分布 | Kelly 仓位决策与系统响应 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 潜伏期 (Latent Edge)** | 微观景气指标 ($N_{raw}$) 开始探底回升，但研报 ($N_{prof}$) 与大V讨论 ($N_{hype}$) 极冷。 | $EGI \approx 1.0$<br>(事实向上，预期在冰点) | 估值处于绝对底部，无人问津。 | **试探建仓 (轻仓)**<br>防止因缺乏“催化剂”陷入时间成本耗损，系统处于侦听与警觉状态。 |
| **2. 黄金击球区 (Golden Sweet Spot)** | 专业投研 ($N_{prof}$) 确认趋势并温和发酵，微观 facts 持续验证，但散户大V ($N_{hype}$) 尚未狂热。 | $EGI \in [0.3, 0.8]$<br>(非对称定价的黄金期) | 趋势已确立，筹码从左侧向专业右侧传导，安全边际极高。 | **仓位全开 (Kelly Max)**<br>系统启动 Sovereign Vision 增益，捕获估值与基本面共振的戴维斯双击波段。 |
| **3. 情绪泡沫期 (Sentiment Hype)** | 专业共识完全透支，雪球大V爆款频出，散户狂热追涨 ($N_{hype}$ 爆发)，但微观指标 ($N_{raw}$) 滞涨。 | $EGI \in [0.0, 0.2]$<br>(定价差几乎耗尽) | 成交量与换手率爆表，估值已经完全透支了乐观预期。 | **只减不增 (Hold / Scale Down)**<br>严格锁定 Kelly 追加，禁止在狂热期加仓，准备随时平仓出局。 |
| **4. 击鼓传花期 (Peak Cliché)** | 散户大V情绪狂暴，一致预期 EPS 吹泡泡，但底层微观 facts ($N_{raw}$) 已见顶回落。 | $EGI < 0.0$<br>(逻辑防御全面崩溃) | 散户成为最终的“套牢盘”，估值处于历史估值泡沫极值。 | **死亡路径激活 (EXIT / 0%)**<br>Bears 发起致命的 `attack`，Kelly 归零，强制止盈/空仓防御，拒绝接最后一棒。 |

### 4.2 级联博弈算法在 Kelly 仓位中的整合

在 `portfolio_manager.py` 的算法模块中，我们将共识传导阶段直接转化为对 Kelly 原始仓位的**共识熵折现系数 ($\eta$)**：

$$\text{Kelly}_{\text{Adjusted}} = \text{Kelly}_{\text{Raw}} \times \eta$$

其中折现因子 $\eta$ 受到 $EGI$ 与 论证摩擦指数 ($AFI$) 的双重惩罚。当系统检测到处于“情绪泡沫期”或“击鼓传花期”（$EGI \le 0.2$ 且 $AFI$ 走高），$\eta$ 会以非线性指数级迅速衰减至 $0.0$，强制截断投资冲动，从而实现“不接最后一棒”的机器绝对理智。

## 5. 落地路线图 (Actionable Roadmap)

### 阶段 1：底座统一与 API 回退（已在 v0.9.1 中初步奠定）
* **动作**：合并 data_providers.py，规范 Tencent、AkShare、Polymarket 异常回退逻辑。
* **成本**：$0 (全部基于公开免 Key 接口)。

### 阶段 2：共识与预期偏离的自动提取（v10.0 重构第一步）
* **动作**：
  1. 编写个股分析师预期提取器（抓取个股一致预期 EPS 曲线与增长率预测）。
  2. 自动根据这组共识，在 Phase 1 初始化阶段为 Inquisitor 生成 `System:ConsensusCliche`。
* **效果**：使辩论不再流于“空洞的 AI 猜想”，而是刀刀砍向市场已定价的分析师预期差。

### 阶段 3：草根微观子模块的动态注册
* **动作**：
  1. 开放 `plugins/` 目录，允许用户编写轻量个股监测脚本（如抓取某个 HS 海关数据编码、某个现货价格网站）。
  2. 辩论循环在动员阶段自动扫描 `plugins/`，如果发现与标的（如 HJT）相关的插件，直接调起并转化为 `EvidenceNode` 注入辩论。
